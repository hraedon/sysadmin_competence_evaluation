This script performs two distinct operations.

The first part generates an inactive account report. The second part is a destructive loop that removes all group memberships from those same accounts.

There are several critical flaws that make this script unsafe for production:

1. Parameter override: The script defines an $InactivityDays parameter (defaulting to 90), but line 20 immediately overwrites it with 30. Any value passed by the caller is discarded; the effective cutoff is always 30 days.

2. No -WhatIf support: The Remove-ADGroupMember call uses -Confirm:$false, but the script provides no -WhatIf or -ReportOnly mode. There is no way to preview which group memberships would be removed without actually running the removal.

3. No removal logging: There is no log output for the group membership removals. Once the script completes, there is no record of which groups were removed from which users, making a rollback or audit impossible.

4. Null LastLogonDate edge case: Accounts that have never logged in (null LastLogonDate) are skipped by the -lt comparison, while accounts with an old login are cleaned up. This discrepancy is likely unintentional.

I would not run this in production. It needs the hardcoded override removed, -WhatIf support added, and proper logging for all destructive actions.
