SVC-MONITORING — Critical. Domain Admin on a monitoring service account is a critical misconfiguration. The blast radius: monitoring agents poll every host in the domain, query WMI, and typically run as a long-lived service. If the monitoring platform is compromised — or if the 847-day-old password is successfully sprayed — the attacker immediately holds Domain Admin with a service account that by design has persistent, broad network access. The AD note ("required for WMI access") is incorrect. WMI access can be delegated at the WMI namespace level (via wmimgmt.msc or Set-WmiNamespaceSecurity) with no Domain Admin required. This claim should not be accepted at face value.

Minimal remediation: remove Domain Admins immediately. Identify exactly which WMI namespaces InfraPulse queries and delegate access only to those. Rotate the password. Open a change management review to understand why ticket #4892's closure did not trigger a follow-up to remove the Domain Admin membership — this is a process failure, not just a misconfiguration. The misconfiguration was introduced intentionally; the problem is that there was no mechanism to ensure it was removed.

TIER0-ADMIN-RWALSH — Critical. Two distinct attack vectors. First: Tier 0 credential on a Tier 2 workstation. LSASS on WS-RWALSH-01 holds the Tier 0 credential after any interactive session. Any process achieving local admin rights on that workstation — browser exploit, phishing payload, malicious macro — can dump LSASS and harvest the Tier 0 hash. Second: the shared Outlook profile on the Tier 0 account is a documented, intentional practice of using a Domain Admin account to read email. This is a separate and compounding vector. Every email with an attachment or link delivered to that account is a potential credential compromise path for a Tier 0 account.

Minimal remediation: require all Tier 0 activity to occur on a dedicated PAW. Remove the shared Outlook profile from the Tier 0 account; rwalsh should have a separate standard user account for email. The workstation logon may take time to enforce organizationally, but removing the Outlook profile is an immediate action.

HELPDESK-LEAD — High. Password reset delegation at the domain root propagates to all OUs including Domain Controllers. The ability to reset the built-in Administrator account password or the DSRM password on any DC is a Tier 0 capability. An attacker who compromises the HELPDESK-LEAD account can reset the DSRM password on a domain controller and attempt offline AD database access.

Minimal remediation: re-scope the delegation to exclude the Domain Controllers OU, AdminAccounts OU, and any Tier 0 containers. Delegate only on OUs containing standard user and workstation accounts.

SVC-BACKUP — Correctly configured: Backup Operators (least privilege for backup software), interactive logon disabled, logon rights scoped to backup servers, password rotated 23 days ago under enforced policy. The standard every service account should be measured against.
