(2) The most significant anomaly is in the Ticket Flags. The TGT (#0) has the 'pre-authent' flag, but both service tickets (#1 and #2) are missing it. In a modern Active Directory environment using AES-256, service tickets should almost always require pre-authentication. The fact that they were issued without it suggests a configuration weakness or an anomaly in how the KDC is handling these specific service requests. This could be related to the "intermittent authentication issues" reported, as it indicates the Kerberos exchange isn't following the standard hardened path.

(3) Access at 18:04:00 is determined by whether a session is already established. The ticket expires at 18:03:22, but existing SMB sessions (like a mapped drive with open files) often persist because the authentication happened at the start of the session. For a new connection attempt, the workstation would use the TGT (#0) to silently request a new service ticket from the KDC (LABDC01). If the TGT is also expired, the user would be prompted to re-authenticate.
