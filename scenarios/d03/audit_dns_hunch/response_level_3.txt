Based on the DNS configuration section in the artifact, I believe the actual cause is clear: west-dns01 holds a secondary zone for corp.internal that has not received a successful zone transfer since 2026-03-15. The DNS record for app.corp.internal was added on 2026-03-20 (after the server migration to a new IP). Since west-dns01's last successful transfer was 2026-03-15 — five days before the record was created — west-dns01 has no record for app.corp.internal. When WKSTN-042 queries west-dns01, it gets NXDOMAIN, which the browser reports as "Server not found."

The zone transfer failures started 2026-03-16, the day after the last successful transfer. The failure reason logged is "Connection timed out." This is consistent with a firewall rule change blocking TCP 53 (zone transfers require TCP, not just UDP). The firewall change on 2026-03-16 likely blocked TCP 53 from west-dns01 to DC01.

The fix is to restore TCP 53 connectivity from west-dns01 to DC01 so zone transfers can resume. Once the firewall rule is corrected, west-dns01 will eventually sync the missing record.
