I recommend implementing a Just-in-Time (JIT) access solution for SQLSRV-01. Developers will request access via an automated portal, and once approved by an Infrastructure Manager, they will be granted administrative rights for a fixed 4-hour window. This elevation will be automatically revoked by the system to prevent standing privilege. 

To follow least privilege, we should not grant full local admin. Instead, we can add the developers to the 'Event Log Readers' group so they can review the SQL Server Error Logs and Windows Event Logs. For service control, we will use a GPO to grant the dev team specific 'Start, Stop, and Pause' rights for the MSSQLSERVER and SQLSERVERAGENT services. All elevation requests, approvals, and access events will be logged to our central SIEM for auditing.
