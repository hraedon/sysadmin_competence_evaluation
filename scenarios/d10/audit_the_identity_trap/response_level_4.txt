The architecture suffers from a circular identity dependency that undermines the Tier 1 and Tier 2 recovery paths. By syncing production identities to the backup environment via Entra Connect, you've ensured that a Tier 0 (Active Directory) compromise is also an Azure compromise. If the domain is hit by ransomware, the attackers can lock the admins out of the Azure portal, rendering the immutable backups useless despite the bits being "safe."

To fix this, we need an "Identity Moat." I recommend creating cloud-only "break-glass" administrator accounts in the Entra ID tenant that are explicitly excluded from sync and have no dependency on the on-premises directory. These accounts should use a separate MFA method, such as FIDO2 security keys, to ensure they remain accessible even if the corporate phone system or Authenticator app environment is compromised. This ensures that Tier 2 (Azure) remains a viable, high-speed recovery path even if Tier 3 (Tape) is the only truly air-gapped fallback.
