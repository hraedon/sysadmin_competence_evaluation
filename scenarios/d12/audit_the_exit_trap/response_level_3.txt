The script is failing because of 'set -e'. If the grep command for 'ERROR' doesn't find anything in the logs, it returns a status of 1. Because errexit is enabled, this 1 is treated as a script failure, so it exits immediately and skips the database vacuum. To fix this, I would change that line to '... | xargs grep "ERROR" || true' so it doesn't fail the whole script when no errors are found.
