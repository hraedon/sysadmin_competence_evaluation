The cron service is currently inactive and dead, which explains why the scheduled tasks are failing. The 'enabled' status just means it is set to start automatically when the server boots up, but it doesn't mean it's running right now. In this case, it's loaded and enabled but inactive. I would start the service using 'systemctl start cron' and then run 'systemctl status cron' again to make sure it transitioned to an active state.
