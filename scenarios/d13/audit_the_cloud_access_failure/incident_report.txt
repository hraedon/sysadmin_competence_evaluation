Incident: INC-6108
Priority: High
Opened: 2026-03-17 09:03
Reported by: DataOps team lead

Title: Azure Storage unreachable from on-premises — broken since Monday morning

Description:
The nightly ETL pipeline that reads from our Azure Storage account (stgproddata01)
started failing at 01:00 Monday (2026-03-16). All on-premises job runners report
"connection refused" or "access denied" when attempting to connect to
stgproddata01.blob.core.windows.net. Azure-hosted jobs connecting to the same storage
account continue to work normally. No changes were made to the pipeline or to Azure
infrastructure over the weekend, to our knowledge.

Update 10:30 — network team response:
"We ran a DNS cleanup project over the weekend (2026-03-14 to 2026-03-15). We
audited and removed several conditional forwarder rules on our on-premises DNS
servers that were identified as stale or redundant. One of the rules removed was
a forwarder for blob.core.windows.net pointing to 168.63.129.16.

We did not make any changes in Azure — no NSG changes, no firewall changes, no
routing changes. This is not a network infrastructure issue on our end. The Azure
storage configuration should be reviewed."
