David,

"Compliant at the time of deployment" is not the same as "compliant." The
corporate security policy defines baseline compliance as a continuous
requirement, not a snapshot. A server that drifts out of compliance two weeks
after deployment is a non-compliant server — it just wasn't caught yet.

Standard practice for any provisioning project is to enroll new servers in
our SCCM and Defender for Endpoint environment so compliance is maintained
automatically. I'm not aware of any project where we've accepted point-in-time
compliance as meeting this requirement.

If Infrastructure's provisioning checklist doesn't include monitoring enrollment,
that's a process gap we need to close — but these servers still need
remediation regardless.

Priya
